## Overview

The Clipboard tool provides read and write access to the Windows system clipboard. Use it to copy text data, retrieve clipboard contents, or facilitate data transfer between applications.

## Function Signature

```python
def clipboard_tool(
    mode: Literal["get", "set"],
    text: str | None = None,
    ctx: Context = None
) -> str
```

## Parameters

<ParamField path="mode" type="string" required>
  The clipboard operation mode:
  - `"get"`: Read the current clipboard content
  - `"set"`: Write text to the clipboard
</ParamField>

<ParamField path="text" type="string" default="None">
  The text to copy to the clipboard.
  
  **Required when:** `mode="set"`
  
  **Ignored when:** `mode="get"`
</ParamField>

## Return Value

### get mode
Returns the current clipboard content:
```
Clipboard content:
[clipboard text]
```

If clipboard is empty or contains non-text data:
```
Clipboard is empty or contains non-text data.
```

### set mode
Returns confirmation with preview (first 100 characters):
```
Clipboard set to: [text preview]...
```

## Usage Examples

<CodeGroup>

```python Read Clipboard
# Get current clipboard content
result = clipboard_tool(mode="get")
# Returns: "Clipboard content:\nSome text that was copied"
```

```python Copy Text to Clipboard
# Set clipboard to specific text
result = clipboard_tool(
    mode="set",
    text="Hello, World!"
)
# Returns: "Clipboard set to: Hello, World!"
```

```python Copy Long Text
# Copy longer text (shows preview in response)
result = clipboard_tool(
    mode="set",
    text="This is a very long piece of text that will be truncated in the confirmation message but fully copied to clipboard"
)
# Returns: "Clipboard set to: This is a very long piece of text that will be truncated in the confirmation message but fu..."
```

```python Copy Multi-line Text
# Copy text with line breaks
result = clipboard_tool(
    mode="set",
    text="Line 1\nLine 2\nLine 3"
)
```

</CodeGroup>

## Supported Data Types

| Data Type | get mode | set mode |
|-----------|----------|----------|
| Plain text (Unicode) | ✅ Supported | ✅ Supported |
| Rich text (RTF) | ❌ Returns "non-text data" | ❌ Not supported |
| Images | ❌ Returns "non-text data" | ❌ Not supported |
| Files | ❌ Returns "non-text data" | ❌ Not supported |

<Note>
The tool only handles Unicode text (`CF_UNICODETEXT` format). Other formats (images, files, formatted text) are not supported.
</Note>

## Best Practices

1. **Wait after copy operations**
   ```python
   shortcut_tool(shortcut="ctrl+c")
   wait_tool(duration=1)  # Give time for clipboard update
   clipboard_tool(mode="get")
   ```

2. **Verify clipboard state before pasting**
   ```python
   # Check clipboard before paste
   current = clipboard_tool(mode="get")
   if "expected content" in current:
       shortcut_tool(shortcut="ctrl+v")
   ```

3. **Handle empty clipboard gracefully**
   ```python
   result = clipboard_tool(mode="get")
   if "empty" in result or "non-text" in result:
       # Clipboard doesn't contain usable text
       pass
   ```

4. **Preserve original clipboard when needed**
   ```python
   # Save original clipboard
   original = clipboard_tool(mode="get")
   
   # Do temporary clipboard operations
   clipboard_tool(mode="set", text="temporary")
   # ... operations ...
   
   # Restore original
   clipboard_tool(mode="set", text=original.replace("Clipboard content:\n", ""))
   ```

## Error Handling

Common errors and solutions:

- **"text parameter required for set mode"**: Provide the `text` parameter when using `mode="set"`
- **"Clipboard is empty"**: No text is currently copied; copy something first
- **"non-text data"**: Clipboard contains an image, file, or formatted content (not supported)
- **"Error managing clipboard"**: Windows clipboard access failed; try again

## Security Considerations

<Warning>
**Sensitive Data**

The clipboard may contain sensitive information:
- Passwords copied from password managers
- API keys or tokens
- Personal information

Be cautious when reading or logging clipboard content.
</Warning>

## Limitations

- **Text only**: Only supports Unicode text format
- **No history**: Cannot access clipboard history (Windows 10/11 clipboard history not supported)
- **Single item**: Can only read/write the most recent clipboard item
- **No format preservation**: Rich text formatting is lost (returns "non-text data")
