## Overview

The **Shortcut** tool executes keyboard shortcuts using key combinations. It's essential for quick actions like copy/paste, window management, and accessing system features that don't have direct UI buttons.

<Tip>
Shortcut supports all standard Windows keyboard shortcuts including Ctrl, Alt, Win, and Shift modifiers.
</Tip>

## Function Signature

```python
def shortcut_tool(shortcut: str, ctx: Context = None)
```

## Parameters

<ParamField path="shortcut" type="string" required>
  Keyboard shortcut as a string with keys separated by `+`. Case-insensitive. Examples: `"ctrl+c"`, `"alt+tab"`, `"win+r"`.
</ParamField>

## Supported Modifier Keys

- `ctrl` - Control key
- `alt` - Alt key  
- `shift` - Shift key
- `win` - Windows key (also accepts `windows`)

Modifiers can be combined with regular keys and with each other.

## Usage Examples

### Copy Selected Text

```json
{
  "shortcut": "ctrl+c"
}
```

Copies selected text or files to clipboard.

### Paste from Clipboard

```json
{
  "shortcut": "ctrl+v"
}
```

Pastes clipboard contents at current cursor position.

### Select All

```json
{
  "shortcut": "ctrl+a"
}
```

Selects all content in the current context (text field, file list, etc.).

### Undo Last Action

```json
{
  "shortcut": "ctrl+z"
}
```

Undo the last action in the active application.

### Redo Last Action

```json
{
  "shortcut": "ctrl+y"
}
```

Redo the previously undone action.

### Switch Applications

```json
{
  "shortcut": "alt+tab"
}
```

Switches to the next application in the task switcher. Note: This opens the switcher but doesn't select an app. Use the **App** tool for precise window switching.

### Open Run Dialog

```json
{
  "shortcut": "win+r"
}
```

Opens the Windows Run dialog for launching applications or opening locations.

### Open Start Menu

```json
{
  "shortcut": "win"
}
```

Opens the Windows Start menu.

### Open Task Manager

```json
{
  "shortcut": "ctrl+shift+esc"
}
```

Directly opens Windows Task Manager.

### Minimize All Windows

```json
{
  "shortcut": "win+d"
}
```

Shows the desktop by minimizing all windows.

### Lock Screen

```json
{
  "shortcut": "win+l"
}
```

Locks the computer and shows the lock screen.

### Save File

```json
{
  "shortcut": "ctrl+s"
}
```

Saves the current document or file.

### Close Window

```json
{
  "shortcut": "alt+f4"
}
```

Closes the active window.

### Find/Search

```json
{
  "shortcut": "ctrl+f"
}
```

Opens the find/search dialog in most applications.

### Refresh Page/Window

```json
{
  "shortcut": "f5"
}
```

Refreshes the current page or view.

### Take Screenshot

```json
{
  "shortcut": "win+shift+s"
}
```

Opens Windows Snipping Tool for screenshots.

### Print

```json
{
  "shortcut": "ctrl+p"
}
```

Opens the print dialog.

### New Window/Tab

```json
{
  "shortcut": "ctrl+n"
}
```

Opens a new window (varies by application).

### Open File

```json
{
  "shortcut": "ctrl+o"
}
```

Opens the file browser dialog in most applications.

## Common Patterns

### Copy-Paste Workflow

1. Select content (with **Click** and drag, or `ctrl+a`)
2. Call **Shortcut** with `"ctrl+c"` to copy
3. Position cursor at destination
4. Call **Shortcut** with `"ctrl+v"` to paste

### Launch Application via Run Dialog

1. Call **Shortcut** with `"win+r"` to open Run
2. Call **Wait** with `duration: 1`
3. Call **Type** with the application name (e.g., `"notepad"`)
4. Call **Type** with `press_enter: true` or **Shortcut** with `"enter"`

### Save Document

1. Ensure the document window is focused
2. Call **Shortcut** with `"ctrl+s"`
3. If "Save As" dialog appears, use **Type** to enter filename
4. Call **Shortcut** with `"enter"` to confirm

### Select All and Replace

1. Call **Shortcut** with `"ctrl+a"` to select all text
2. Call **Type** with new content (automatically replaces selection)

### Search in Application

1. Call **Shortcut** with `"ctrl+f"` to open find dialog
2. Call **Type** with search query
3. Call **Shortcut** with `"enter"` or `"f3"` to find next

## Troubleshooting

<AccordionGroup>
  <Accordion title="Shortcut has no effect">
    Possible causes:
    - Wrong window is focused (use **App** tool to switch focus first)
    - Application doesn't support that shortcut
    - Application is running with elevated privileges (run Windows-MCP elevated)
    - Shortcut is disabled in application settings
  </Accordion>

  <Accordion title="Invalid key name error">
    Check your shortcut string format:
    - Use `+` to separate keys: `"ctrl+c"` not `"ctrl c"`
    - Use lowercase or uppercase (case-insensitive): both `"Ctrl+C"` and `"ctrl+c"` work
    - Check for typos in key names
  </Accordion>

  <Accordion title="Alt+Tab doesn't switch to specific window">
    The `alt+tab` shortcut only opens the task switcher. For precise window switching:
    - Use the **App** tool with `mode: 'switch'` and `name: 'Window Title'`
    - Or use **Snapshot** to find the window and **Click** on it
  </Accordion>

  <Accordion title="Shortcut triggers wrong action">
    Some applications override standard shortcuts:
    - Check the application's keyboard shortcuts in its settings
    - Some apps use different shortcuts (e.g., `ctrl+q` vs `alt+f4` to quit)
    - Try application-specific shortcuts instead
  </Accordion>

  <Accordion title="Modifier keys stick">
    If modifier keys appear stuck after a shortcut:
    - This is rare but can happen if execution is interrupted
    - Send individual modifier keys to release them
    - Restart the application if necessary
  </Accordion>

  <Accordion title="Can't send single key press">
    For single keys without modifiers:
    - Use the **Type** tool instead: `{"text": "a"}`
    - **Shortcut** works but is intended for combinations
    - Function keys work: `{"shortcut": "f5"}`
  </Accordion>

  <Accordion title="Windows key shortcuts disabled">
    Group Policy or system settings may disable Windows key shortcuts:
    - Check system Group Policy settings
    - Some enterprise/kiosk configurations block Win key
    - Try equivalent actions through **App** or **PowerShell** tools
  </Accordion>
</AccordionGroup>
