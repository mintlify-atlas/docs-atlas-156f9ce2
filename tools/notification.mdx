## Overview

The Notification tool sends Windows toast notifications to alert users with a title and message. This is useful for remote operations, task completion alerts, or drawing user attention to important events.

## Function Signature

```python
def notification_tool(
    title: str,
    message: str,
    ctx: Context = None
) -> str
```

## Parameters

<ParamField path="title" type="string" required>
  The notification title. Displayed in bold at the top of the toast.
  
  **Best practices:**
  - Keep it short and clear (max ~50 characters)
  - Summarize the notification purpose
  - Use title case or sentence case
  
  **Examples:** `"Task Completed"`, `"Download Ready"`, `"System Alert"`
</ParamField>

<ParamField path="message" type="string" required>
  The notification message body. Displayed below the title.
  
  **Best practices:**
  - Provide context or details (max ~200 characters recommended)
  - Use clear, actionable language
  - Include relevant information (file name, status, next steps)
  
  **Examples:** `"Your document has been saved to Downloads"`, `"3 files processed successfully"`
</ParamField>

## Return Value

Returns a confirmation string from the desktop service, typically:
```
Notification sent successfully
```

Or an error message if the notification failed:
```
Error sending notification: [error details]
```

## Notification Behavior

<AccordionGroup>

<Accordion title="Display Duration">
  Windows toast notifications typically:
  - Display for ~5-10 seconds by default
  - Can be dismissed by clicking the X button
  - Remain in Action Center (Windows 10/11) after dismissal
  - Auto-dismiss if user doesn't interact
</Accordion>

<Accordion title="User Focus">
  Notifications appear:
  - In the bottom-right corner (default Windows position)
  - Even when user is working in other applications
  - Above most windows but below full-screen applications
  - In the Action Center for later review
</Accordion>

<Accordion title="Sound and Priority">
  - Standard notifications play the system notification sound
  - Volume follows Windows notification settings
  - Priority cannot be customized (always standard toast)
</Accordion>

</AccordionGroup>

## Best Practices

1. **Use clear, actionable language**
   ```python
   # ✅ Good: Clear and specific
   notification_tool(
       title="Export Complete",
       message="5 reports exported to C:\\Reports\\2024"
   )
   
   # ❌ Bad: Vague and unhelpful
   notification_tool(
       title="Done",
       message="The thing finished"
   )
   ```

2. **Keep it concise**
   ```python
   # ✅ Good: Brief and informative
   notification_tool(
       title="Scan Complete",
       message="Found 3 issues. Check log for details."
   )
   
   # ❌ Bad: Too long
   notification_tool(
       title="Scan Complete",
       message="The system scan has finished running and found 3 issues that you should review by checking the log file located at..."
   )
   ```

3. **Match tone to urgency**
   ```python
   # Critical issues
   notification_tool(
       title="⚠️ Critical Error",
       message="Database connection lost. Immediate action required."
   )
   
   # Routine updates
   notification_tool(
       title="Update Available",
       message="A new version is ready to install"
   )
   ```

4. **Include next steps when appropriate**
   ```python
   notification_tool(
       title="Approval Required",
       message="Review and approve 3 pending requests in the admin panel"
   )
   ```

## Workflow Examples

        title="Copy Failed",

## When NOT to Use Notifications

❌ **Avoid notifications for:**
- Every minor action (creates notification spam)
- Information the user already sees on screen
- Non-urgent status updates
- Frequent repetitive events

✅ **Use notifications for:**
- Background task completion
- Remote/unattended operations
- Important alerts or errors
- Events requiring user attention

## Platform Requirements

- **Windows 10+**: Full toast notification support
- **Windows 8.1**: Basic toast support
- **Windows 7**: May not display or fall back to system tray

## Limitations

- **No interactive buttons**: Cannot add action buttons to notifications
- **No images**: Cannot include custom images or icons
- **No sound customization**: Uses default system notification sound
- **No priority levels**: All notifications have standard priority
- **No scheduling**: Notifications send immediately

## Error Handling

Common errors and solutions:

- **"Error sending notification"**: Windows notification service may be disabled; check Windows settings
- **Notification doesn't appear**: Check that Focus Assist is not enabled (blocks notifications)
- **No sound**: Verify notification sounds are enabled in Windows sound settings

## Accessibility Considerations

- **Screen readers**: Windows toast notifications are read by screen readers
- **High contrast**: Notifications respect Windows high contrast themes
- **Duration**: Users with disabilities may need more time; notifications remain in Action Center
