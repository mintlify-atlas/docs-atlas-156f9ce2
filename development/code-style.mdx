Windows-MCP follows consistent code style guidelines to ensure readability and maintainability. We use [Ruff](https://github.com/astral-sh/ruff) for automated formatting and linting.

## Formatting and Linting

We use **Ruff** as our formatter and linter. It's fast, Python 3.13 compatible, and replaces multiple tools (Black, Flake8, isort).

### Configuration

Ruff is configured in `pyproject.toml`:

```toml
[tool.ruff]
line-length = 100
target-version = "py313"

[tool.ruff.lint]
select = ["E", "F", "W"]
ignore = ["E501"]

[tool.ruff.lint.per-file-ignores]
"src/windows_mcp/uia/*.py" = ["F403", "F405"]
```

### Running Ruff

<Steps>
  <Step title="Format Code">
    Format all Python files to match the project's style:
    
    ```bash
    ruff format .
    ```
  </Step>
  
  <Step title="Lint Code">
    Check for linting issues:
    
    ```bash
    ruff check .
    ```
  </Step>
  
  <Step title="Auto-fix Issues">
    Automatically fix linting issues where possible:
    
    ```bash
    ruff check --fix .
    ```
  </Step>
</Steps>

<Tip>
  Always run `ruff format .` and `ruff check .` before committing your changes.
</Tip>

## Style Guidelines

### Line Length

**Maximum line length: 100 characters**

Break long lines to improve readability:

```python
# Good
def click_tool(
    loc: list[int],
    button: Literal['left', 'right', 'middle'] = 'left',
    clicks: int = 1
) -> str:
    pass

# Avoid
def click_tool(loc: list[int], button: Literal['left', 'right', 'middle'] = 'left', clicks: int = 1) -> str:
    pass
```

### Quotes

**Use double quotes for strings:**

```python
# Good
message = "Click performed successfully"
path = "C:\\Users\\Documents"

# Avoid
message = 'Click performed successfully'
path = 'C:\\Users\\Documents'
```

### Naming Conventions

Follow PEP 8 naming conventions:

### Type Hints

**Always add type annotations to function signatures:**

```python
# Good
def click_tool(
    loc: list[int],
    button: Literal['left', 'right', 'middle'] = 'left',
    clicks: int = 1
) -> str:
    """Click on UI elements at specific coordinates."""
    if len(loc) != 2:
        raise ValueError("Location must be a list of exactly 2 integers [x, y]")
    return "Click performed"

# Avoid (no type hints)
def click_tool(loc, button='left', clicks=1):
    if len(loc) != 2:
        raise ValueError("Location must be a list of exactly 2 integers [x, y]")
    return "Click performed"
```

### Docstrings

**Use Google-style docstrings for all public functions and classes:**

```python
def click_tool(
    loc: list[int],
    button: Literal['left', 'right', 'middle'] = 'left',
    clicks: int = 1
) -> str:
    """Click on UI elements at specific coordinates.
    
    Args:
        loc: List of [x, y] coordinates to click
        button: Mouse button to use (left, right, or middle)
        clicks: Number of clicks (1=single, 2=double, 3=triple)
    
    Returns:
        Confirmation message describing the action performed
    
    Raises:
        ValueError: If loc doesn't contain exactly 2 integers
    """
    if len(loc) != 2:
        raise ValueError("Location must be a list of exactly 2 integers [x, y]")
    # Implementation...
    return f"Clicked at ({loc[0]}, {loc[1]}) with {button} button"
```

**Docstring structure:**

1. **Summary line**: Brief one-line description
2. **Args**: Document all parameters (name and description)
3. **Returns**: Describe the return value
4. **Raises**: List exceptions that can be raised

### Helper Function Pattern

From `src/windows_mcp/desktop/service.py:50`:

```python
def _escape_text_for_sendkeys(text: str) -> str:
    """Escape special characters so uia.SendKeys types them correctly."""
    result = []
    for ch in text:
        if ch == "{":
            result.append("{{}")  
        elif ch == "}":
            result.append("{}}")  
        elif ch == "\n":
            result.append("{Enter}")
        elif ch == "\t":
            result.append("{Tab}")
        elif ch == "\r":
            continue
        else:
            result.append(ch)
    return "".join(result)
```

## Comments

### When to Comment

**Do comment:**

- Complex algorithms or logic
- Non-obvious decisions or workarounds
- Important edge cases or limitations
- TODO items or known issues

```python
# Screenshots are capped to 1920x1080 for token efficiency
max_width = 1920
max_height = 1080

# Browser detection triggers special DOM extraction mode
if window.browser in [Browser.CHROME, Browser.EDGE, Browser.FIREFOX]:
    use_dom = True
```

**Don't comment:**

- Obvious code that's self-explanatory
- Redundant explanations of what the code does

```python
# Bad: Comment is obvious
# Increment counter by 1
counter += 1

# Good: Self-explanatory code needs no comment
counter += 1
```

## Imports

**Group imports in the following order:**

1. Standard library imports
2. Related third-party imports
3. Local application/library imports

```python
# Standard library
from typing import Literal
from time import sleep
import logging

# Third-party
from PIL import Image
from fuzzywuzzy import process
import win32gui

# Local
from windows_mcp.desktop.views import DesktopState, Window
from windows_mcp.tree.service import Tree
import windows_mcp.uia as uia
```

## Error Handling

**Raise appropriate exceptions with clear messages:**

```python
# Good
if len(loc) != 2:
    raise ValueError("Location must be a list of exactly 2 integers [x, y]")

if not isinstance(clicks, int) or clicks < 1:
    raise ValueError("Clicks must be a positive integer")

# Avoid generic exceptions
if len(loc) != 2:
    raise Exception("Invalid location")
```
