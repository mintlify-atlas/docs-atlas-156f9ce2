Get Windows-MCP running in your MCP client and start automating Windows tasks with AI.

## Prerequisites

Before you begin, ensure you have:

- **Windows 7 or later** (Windows 10/11 recommended)
- **Python 3.13+** installed
- **UV package manager** (install with `pip install uv` or `curl -LsSf https://astral.sh/uv/install.sh | sh`)
- An MCP-compatible client (Claude Desktop, Perplexity Desktop, Gemini CLI, etc.)

<Note>
When installing Windows-MCP for the first time, the initial setup may take 1-2 minutes to install dependencies. If the server times out on first run, simply restart your MCP client.
</Note>

## Installation

<Steps>
  <Step title="Install an MCP Client">
    Download and install an MCP-compatible client. We'll use [Claude Desktop](https://claude.ai/download) as an example.

    ```bash
    # Claude Desktop is available for download at claude.ai/download
    ```
  </Step>

  <Step title="Configure Windows-MCP">
    Add Windows-MCP to your MCP client's configuration file.

    **For Claude Desktop**, edit `claude_desktop_config.json`:

    <CodeGroup>
    ```json PyPI Installation (Recommended)
    {
      "mcpServers": {
        "windows-mcp": {
          "command": "uvx",
          "args": [
            "windows-mcp"
          ]
        }
      }
    }
    ```

    ```json Source Installation
    {
      "mcpServers": {
        "windows-mcp": {
          "command": "uv",
          "args": [
            "--directory",
            "C:\\path\\to\\Windows-MCP",
            "run",
            "windows-mcp"
          ]
        }
      }
    }
    ```
    </CodeGroup>

    <Note>
    **Claude Desktop (Windows Store/MSIX):** The MSIX-packaged version virtualizes `%APPDATA%`. Your config file is located at:
    
    `%LOCALAPPDATA%\Packages\Claude_pzs8sxrjxfjjc\LocalCache\Roaming\Claude\claude_desktop_config.json`
    
    The "Edit Config" button may open the wrong file. Use the full path above instead.
    </Note>

    <Warning>
    **MSIX users:** Electron apps don't inherit PATH. Use the **full absolute path** to `uv.exe`:

    ```json
    {
      "mcpServers": {
        "windows-mcp": {
          "command": "C:\\Users\\YourUsername\\.local\\bin\\uv.exe",
          "args": ["windows-mcp"]
        }
      }
    }
    ```

    Find `uv.exe` by running `where uv` in PowerShell. Save the config file as **UTF-8 without BOM** (PowerShell's `Set-Content -Encoding UTF8` adds a BOM that breaks JSON parsing).
    </Warning>
  </Step>

  <Step title="Launch Your MCP Client">
    Open Claude Desktop (or your chosen MCP client). Windows-MCP will start automatically.

    Look for the MCP server indicator in your client. In Claude Desktop, you'll see a 🔌 icon when Windows-MCP is connected.
  </Step>

  <Step title="Verify Installation">
    Test Windows-MCP with a simple command:

    **In Claude Desktop, type:**
    ```
    Take a screenshot and tell me what's on my desktop
    ```

    Claude will use the `Snapshot` tool to capture your desktop state and describe what it sees.
  </Step>
</Steps>

## Your First Automation

Let's try a more advanced example that showcases Windows-MCP's capabilities:

<Steps>
  <Step title="Open Notepad">
    **Ask your AI client:**
    ```
    Open Notepad and type "Hello from Windows-MCP!"
    ```

    Behind the scenes, Windows-MCP will:
    1. Use the `App` tool to launch Notepad
    2. Use the `Snapshot` tool to locate the text area
    3. Use the `Type` tool to enter the text
  </Step>

  <Step title="Take a Screenshot">
    **Ask your AI client:**
    ```
    Take a screenshot with the cursor visible
    ```

    The AI will use:
    ```json
    Snapshot tool with use_vision=True
    ```

    You'll receive both the screenshot and a structured list of all interactive UI elements.
  </Step>

  <Step title="Navigate Your File System">
    **Ask your AI client:**
    ```
    List all files on my Desktop
    ```

    The AI will use the `FileSystem` tool in list mode to enumerate your Desktop files.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Server timeout on first launch">
    **Issue:** Windows-MCP times out during initial installation.
    
    **Solution:** This is normal. Dependencies are being installed (can take 1-2 minutes). Restart your MCP client after the first timeout.
  </Accordion>

  <Accordion title="MSIX spawn ENOENT error">
    **Issue:** Claude Desktop (Windows Store) shows `spawn uvx ENOENT`.
    
    **Solution:** Use the full absolute path to `uv.exe` instead of `uvx`:
    
    ```json
    {
      "command": "C:\\Users\\YourUsername\\.local\\bin\\uv.exe",
      "args": ["windows-mcp"]
    }
    ```
    
    Find your `uv.exe` path: `where uv` in PowerShell.
  </Accordion>

  <Accordion title="Config file not found">
    **Issue:** Can't find `claude_desktop_config.json`.
    
    **Solution:**
    - **Standard install:** `%APPDATA%\Claude\claude_desktop_config.json`
    - **MSIX install:** `%LOCALAPPDATA%\Packages\Claude_pzs8sxrjxfjjc\LocalCache\Roaming\Claude\claude_desktop_config.json`
  </Accordion>

  <Accordion title="JSON parsing error">
    **Issue:** Config file throws parsing errors.
    
    **Solution:** Ensure the file is saved as **UTF-8 without BOM**. PowerShell's `Set-Content -Encoding UTF8` adds a BOM. Use a text editor like VS Code or Notepad++ to save without BOM.
  </Accordion>

  <Accordion title="App tool not working with non-English Windows">
    **Issue:** App tool fails on non-English Windows.
    
    **Solution:** Windows-MCP requires English as the default Windows language. If you use another language, disable the App tool in your configuration or switch Windows to English.
  </Accordion>
</AccordionGroup>

## Disable Telemetry (Optional)

Windows-MCP collects anonymized usage data to improve the server. No personal information, tool arguments, or outputs are tracked.

To disable telemetry, add an environment variable to your configuration:

```json
{
  "mcpServers": {
    "windows-mcp": {
      "command": "uvx",
      "args": ["windows-mcp"],
      "env": {
        "ANONYMIZED_TELEMETRY": "false"
      }
    }
  }
}
```
